## LittleLOOP

Простой планировщик задач для микроконтроллеров, предоставляющий минимальный функционал для поочередного запуска множества задач, схожий с RTOS, только в разы проще. Весь проект написан на чистом C без внешних зависимостей, и по умолчанию умеет работать с фреймворками ch32fun и Arduino.

Пример кода Arduino-кода, мигающий светодиодом каждые 500мс (при tickrate = 1000)
```sh
#include "Arduino.h"
#include <littleloop.h>

// можно использовать статические переменные
static bool led_state = 0;
void blink_led() {
  led_state = !led_state; // Инвертируем состояние светодиода
  digitalWrite(LED_BUILTIN, led_state);
}

void setup() {
  pinMode(LED_BUILTIN, OUTPUT);

  // возвращает индекс задачи, период выполнения 1 раз в 500 тиков
  int32_t led_task = lloop_add_task(blink_led, 500);
  // по умолчанию задачи создаются активными, но их можно явно активировать/деактивировать
  lloop_enable_task(led_task);

  //lloop_add_task(blink_led_2, 100);
  //lloop_add_task(blink_led_3, 5000);
  //lloop_add_task(blink_led_4, 400);

  // планировщик полностью блокирует выполнение программы!
  lloop_start_auto_scheduler();
}

void loop() {
  // Тут пусто, так как планировщик блокирует выполнение.
}

```
Преимущества библиотеки:
- Отсутствие внешних зависимостей
- Отсутствие прерываний
- Быстрое портирование на новое железо
- **Только статическая аллокация памяти**
- Нет необходимости указывать размер стека для новых задач
- Крайне малый размер, работает на самых слабых микроконтроллерах
- Динамическое включение/отключение задач даже во время работы планировщика
- Работа с C и C++ кодом
- Свободная лицензия BSD 3 clause
- Возможность настройки через loopconfig.h

Минусы:
- **Отсутствие паралеллизма**
- LOOP_AUTO_TICK_RATE должен быть кратным 1000 (50, 100, 125, 200, 250, 500, 1000 и т.д.)
- Также LOOP_AUTO_TICK_RATE не может превышать 1000

Без портирования код автоматического планировщика не будет доступен, но его все ещё можно вызывать вручную. Ниже пример.
```sh
#include "Arduino.h"
#include <littleloop.h>

static bool led_state = 0;
void blink_led() {
  led_state = !led_state; // Инвертируем состояние светодиода
  digitalWrite(LED_BUILTIN, led_state);
}

void setup() {
  pinMode(LED_BUILTIN, OUTPUT);

  int32_t led_task = lloop_add_task(blink_led, 500);
  lloop_enable_task(led_task);

  for (;;) {
    lloop_step_tick();
    delay(1);
  }
}

void loop() {
  
}
```

Для работы автопланировщика на новом микроконтроллере необходимо только добавить в loopconfig.h всего несколько переменных, вот пример для ch32fun
```sh
// Общие настройки
#define LOOP_AUTO_TICK_RATE 1000 // 1000 тиков в секунду
#define TASK_QUEUE_SIZE 50 // максимальное количество задач
#define DEFAULT_TASK_STATE enabled

// Тут используется счетчик инструкций процессора, схожий функционал доступен на чипах всех производителей
#define LOOP_SYSTICK_FUNC SysTick->CNT
// Количество инструкций выполняемых за 1мс, можно посчитать как (Fclk / 1000)
#define LOOP_SYSTICK_MS_DELIMETER DELAY_MS_TIME
// Включение кода для планировщика
#define LOOP_ENABLE_AUTO_SCHEDULER
```
